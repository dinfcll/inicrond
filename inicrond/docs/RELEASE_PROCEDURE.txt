This is my release procedure.

Before tagging the files :

* check the patch entry and check if the schema is equal to it.
* try a clean install.
* publish languages files.
* patch a production dataset in local.
	source <sql_file>
*update the version constants.

After before tagging the files (of course.):

* update the tag file.
* commit with release announcement
* tag the files
	cvs tag <tag_name> *
	
* export the files
	sh export_cvs.sh <tag_name>
	
* sourceforge release
* freshmeat release
